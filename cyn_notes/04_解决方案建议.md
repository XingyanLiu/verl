# 解决方案建议

## 方案一：调整梯度裁剪阈值（推荐，详见 05、06 号文档）

> **更新**：经过对 AdamW 优化器的详细理论分析（见 `05_loss_scale对AdamW的影响分析.md`），
> 推荐**优先调整梯度裁剪阈值**而非学习率。AdamW 的自适应步长对均匀 loss 缩放近似不变，
> 但梯度裁剪行为会因缩放发生显著变化。详细迁移方案见 `06_迁移策略.md`。

```yaml
actor_rollout_ref:
  actor:
    grad_clip: 0.03125  # = 旧 grad_clip (1.0) / K (32)
```

如果确实需要调学习率（例如旧版本训练从未触发梯度裁剪），可以等比放大：

```
新学习率 ≈ 旧学习率 × K
```

**注意**：如果同时改变了 batch size 或并行度，需要综合考虑。

## 方案二：使用 seq-mean-token-mean 模式

将 `loss_agg_mode` 从 `token-mean` 改为 `seq-mean-token-mean`：

```yaml
actor_rollout_ref:
  actor:
    loss_agg_mode: "seq-mean-token-mean"
```

`seq-mean-token-mean` 模式的公式：
```
seq_losses[j] = Σ_i(loss[j,i] * mask[j,i]) / Σ_i(mask[j,i])   # 每个序列内 token-mean
loss = Σ_j(seq_losses[j] * seq_mask[j]) / global_batch_size * dp_size   # 序列间 mean
```

这个模式：
- 每个序列的 loss 贡献与其长度无关
- 与 DAPO 论文的原始意图更接近
- loss 量级更稳定（不受总 token 数变化影响）

**但需注意**：DAPO 论文实际推荐 token-mean，而非 seq-mean-token-mean。切换模式可能影响训练效果。

---

## 方案三：设置 micro_batch_size_per_gpu=1

将 micro_batch_size_per_gpu 设为 1，使每个 micro-batch 只有一个序列：

```yaml
actor_rollout_ref:
  actor:
    micro_batch_size_per_gpu: 1
```

此时 batch-level token-mean ≈ per-sequence token-mean，loss 量级与旧版本近似。

**缺点**：可能影响 GPU 利用率和训练速度。

---

## 方案四：使用 seq-mean-token-sum-norm 模式

如果想保持 loss 量级稳定且不受 batch 大小影响：

```yaml
actor_rollout_ref:
  actor:
    loss_agg_mode: "seq-mean-token-sum-norm"
    loss_scale_factor: 1024  # 设为固定值，如典型的 response_length
```

`seq-mean-token-sum-norm` 公式：
```
seq_losses[j] = Σ_i(loss[j,i] * mask[j,i])   # 每个序列 token-sum
loss = [Σ_j(seq_losses[j] * seq_mask[j]) / global_batch_size * dp_size] / loss_scale_factor
```

**说明**：`loss_scale_factor` 是一个固定的归一化常数，确保 loss 在不同 response 长度下都保持合理的量级。

---

## 方案五：回退到旧版 compute_policy_loss（不推荐）

当前代码中保留了旧版 `compute_policy_loss` 函数（标记为 `@deprecated`），可以在 experimental 代码中直接调用。但这不推荐因为：
1. 已被标记为废弃，未来可能移除
2. 不支持分布式训练下的正确归一化
3. 缺少 dual-clip 等新特性

---

## 推荐操作步骤

### 快速验证问题
1. 设置 `micro_batch_size_per_gpu=1` 运行一小段训练
2. 对比 pg_loss 量级是否恢复到 v0.3.x 水平
3. 如果恢复，确认问题确实是归一化导致的

### 正式修复
1. **优先方案一**：保持 `token-mean` 模式，等比调整学习率
2. **备选方案二**：切换 `seq-mean-token-mean`，需重新调优超参
3. 监控 `actor/pg_loss`、`actor/ppo_kl`、reward 趋势确认训练正常

### 注意事项
- 仅看 pg_loss 绝对值大小不能判断训练是否正常，更重要的是看 **reward 和准确率的趋势**
- loss 的量级变化 ≠ 梯度的量级变化（梯度在框架内部被正确累积）
- 但如果学习率等超参是基于旧 loss 量级调优的，确实需要调整

---

## 相关配置参数速查

```yaml
actor_rollout_ref:
  actor:
    # Loss 聚合模式
    loss_agg_mode: "token-mean"  # "seq-mean-token-sum" / "seq-mean-token-mean" / "seq-mean-token-sum-norm"

    # DAPO 特有的 clip 参数
    clip_ratio: 0.2
    clip_ratio_low: 0.2      # ε_low
    clip_ratio_high: 0.28    # ε_high
    clip_ratio_c: 3.0        # dual-clip 下界

    # Micro-batch 大小
    micro_batch_size_per_gpu: 4

    # 固定的 loss 归一化因子（仅 seq-mean-token-sum-norm 模式使用）
    loss_scale_factor: null  # 设为固定整数（如 1024）可稳定 loss 量级
```

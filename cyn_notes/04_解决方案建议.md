# 解决方案建议（修正版）

> **⚠️ 重要修正**：之前的方案基于错误前提（v0.7 引入了 K 倍 loss 缩放差异）。
> 经审查确认：**v0.3 和 v0.7 在 dp_actor 路径下的有效梯度完全相同。**
> 不需要为"版本间 loss 缩放差异"调整超参数。
>
> 如果用户在 v0.7 上训练效果不如 v0.3，原因可能来自**算法变更**（见下文），
> 而非 loss 缩放。

## 实际差异来源

v0.3 到 v0.7 的算法变更可能影响训练效果：

| 变更 | 影响 | 建议 |
|------|------|------|
| Dual-clip (`clip_ratio_c = 3.0`) | 对负 advantage 的处理不同 | 如不需要，设 `clip_ratio_c: 100.0` 禁用 |
| 分离 clip_ratio_low/high | 可能与旧版单一 cliprange 不同 | 确保两者设为相同值 |
| Log-ratio clamping (-20, 20) | 大多数情况下无影响 | 通常无需调整 |
| Metric 报告格式变化 | 日志中数值不同 | 理解差异即可，不影响训练 |

---

## 如果确实观测到训练行为差异

### Step 1：排除算法变更的影响

```yaml
actor_rollout_ref:
  actor:
    clip_ratio: 0.2
    clip_ratio_low: 0.2
    clip_ratio_high: 0.2     # 与旧版保持一致
    clip_ratio_c: 100.0       # 禁用 dual-clip
    loss_agg_mode: "token-mean"  # 默认，与旧版等价
```

### Step 2：对比关键指标

运行短训练（50 步），对比：
- `actor/pg_clipfrac`：应与旧版相近
- `actor/ppo_kl`：应与旧版相近
- `actor/grad_norm`：应与旧版相近（两版梯度等价）
- reward 趋势

### Step 3：如果 grad_norm 差异很大

检查是否有其他配置差异（如 batch_size, learning_rate, micro_batch_size 等）。
两个版本的 loss 缩放相同，grad_norm 应该一致。

## 其他有效的配置选项

v0.7 新增的 `loss_agg_mode` 选项本身是有用的功能（不是补偿方案）：

```yaml
actor_rollout_ref:
  actor:
    # 默认模式（与 v0.3 等价）
    loss_agg_mode: "token-mean"

    # 序列级别聚合（可选）
    # loss_agg_mode: "seq-mean-token-mean"  # 每个序列先 token-mean，再 seq-mean

    # DAPO 特有的 clip 参数
    clip_ratio: 0.2
    clip_ratio_low: 0.2      # ε_low
    clip_ratio_high: 0.28    # ε_high (DAPO)
    clip_ratio_c: 3.0        # dual-clip 下界 (DAPO)
```

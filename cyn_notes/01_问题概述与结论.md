# verl 版本升级后 pg_loss 相关问题分析（修正版）

## 问题描述

使用 verl-0.7.0 跑 DAPO RL 训练任务（之前 verl-0.3.x 可正常训练），发现相同设定下 `pg_loss` 的幅度有差异。

## 核心结论（修正版）

> **⚠️ 重要修正**：之前的结论声称 v0.7 引入了新的 loss 全局归一化导致 pg_loss 缩小 K 倍。
> 经仔细审查源码后发现：**v0.3 和 v0.7 在 dp_actor 路径下的 loss 计算完全相同。**
> 两个版本都除以 gradient_accumulation（= K），有效梯度等价。

**v0.3 与 v0.7 的实际关系（dp_actor 路径）：**

| 维度 | v0.3.x | v0.7.0+ (当前) | 相同？ |
|------|--------|----------------|--------|
| pg_loss 聚合 | `masked_mean`（micro-batch 内） | `agg_loss` → 等价于 `masked_mean` | ✅ 相同 |
| 除以 K | `loss / gradient_accumulation` | `loss * (1 / gradient_accumulation)` | ✅ 相同 |
| 有效梯度 | `∇(masked_mean) / K` | `∇(masked_mean) / K` | ✅ 相同 |
| K 的定义 | `ppo_mini_batch_size // ppo_micro_batch_size_per_gpu` | 同左 | ✅ 相同 |

**真正的差异来源：**

1. **算法变更**：v0.7 引入了 dual-clip（`clip_ratio_c`）、分离的 clip_ratio_low/high、log-ratio clamping 等
2. **Metric 报告格式**：v0.3 以 list 格式报告原始 pg_loss；v0.7 以 scalar 格式报告 scaled pg_loss（乘以 1/K）
3. **新功能**：多种 loss 聚合模式（但默认 `token-mean` 与 v0.3 等价）

**Metric 报告差异可能导致日志中看到的数值不同，但不影响实际训练梯度。**

## 关键事实

- K = ppo_mini_batch_size / ppo_micro_batch_size_per_gpu（代码位置：dp_actor.py 第 562-563 行）
- K **不包含** dp_size（GPU 数量）
- 在 dp_actor 路径下，`config.global_batch_info` 为空字典，`agg_loss` 退化为 `masked_mean`
- 两个版本有效梯度完全等价，不需要调整学习率或梯度裁剪来补偿"版本差异"
